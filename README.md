2. Project Overview 
	● Project Name: EMBER STUDIOS
	● Project Description: Ember Studios is a digital art portfolio platform designed to showcase our collective creative works, projects, and artistic growth throughout our college years and future endeavors. The system serves as both an online gallery for public viewing and a management tool for artists to organize, present, and share their artworks in a professional and engaging way
	● Target Users: Who will use this system? 
		○ Public visitors (view artworks) 
		○ Admin users (manage content) 
		○ Artists/members (if applicable)
		○ Gamers/Developers
	● Project Goal: The goal of Ember Studios is to provide a centralized platform for showcasing our creative works and projects in a professional and accessible way. It aims to highlight our skills, creativity, and growth as artists and developers while offering a space where our efforts can be appreciated by a wider audience beyond the classroom.

3. System Functionality & Features  
Public Viewers Features: 
	● Browse gallery of artworks 
	● Filter by category (2D Art, Games, Animation, Photography) 
	● Search artworks by keyword 
	● View individual artwork details 
	● View artist/member profiles 
	● Submit contact form

Admin Features (What you as admin can do): 
	● Secure login system 
	● Dashboard with statistics (total artworks, views, messages) 
	● Upload new artwork (with image, title, description, category) 
	● Edit existing artworks 
	● Delete artworks 
	● Manage categories 
	● View and respond to contact messages 
	
Bonus/Advanced Features: 
	● Like/favorite system 
	● Comment system 
	● Download statistics (resume)
	● Featured artwork slider 
	● Dark mode toggle 
	● Multi-user admin system 
	● Social media sharing 

4. Technology Stack  
Frontend Technologies: 
	● HTML5 - Structure and content 
	● CSS3 - Styling and layout 
	● JavaScript - Interactivity and dynamic behavior 
	● Tailwind CSS (optional) – A utility-first CSS framework used for efficient and responsive design.
	● Icons: Font Awesome, hand-drawn icons

Backend Technologies: 
	● JavaScript (Firebase Integration) – Chosen for its simplicity and compatibility with the frontend, making it easier to style and manage dynamic content.
	● File upload handling - Enables admins to upload and manage artwork images and related content.
	● Authentication system - Provides secure access for admin users to manage site content.
	● Firebase Hosting - Used for deploying and hosting the website with scalability, speed, and reliability.

7. Expected Challenges & Solutions 
Identify potential difficulties and how you'll handle them: 
Technical Challenges: 
	● Challenge: "We've never connected JavaScript to PHP/database before" 
	○ Solution: Research tutorials, watch YouTube guides, ask instructor 
for help 
	● Challenge: "File upload might be complicated" 
	○ Solution: Start with simple file upload, test with small images first, 
research security best practices 
	● Challenge: "Authentication and password security is new to us" 
	○ Solution: Use PHP's password_hash() and password_verify(), study 
prepared statements 

Team Challenges: 
	● Challenge: "Schedules might conflict for group meetings" 
	○ Solution: Use Discord for async communication, set fixed 
weekly meeting times for updates
	● Challenge: "What if someone doesn't contribute equally?" 
	○ Solution: Regular check-ins and follow ups every Wednesday or tanggalin na

Time Management Challenges: 
	● Challenge: "We might fall behind schedule" 
	○ Solution: Have a backup plan (some features not present)
 	● Challenge: "What if the Backup plan fails" 
	○ Solution: Have a backup for a back up plan (simple working website)
 	● Challenge: "Workload from other subjects"
	○ Solution: set schedules for backend developers to cover the task that day, and the other the next day
